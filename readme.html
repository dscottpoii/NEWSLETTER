# The Boat Pro's Newsletter - Boat Newsletter Website A complete, multi-page newsletter website for recreational boat enthusiasts, owners, and prospective buyers. ## Site Structure ### Pages 1. **index.html** - Home page with features overview, latest articles preview, and testimonials 2. **about.html** - About page with team bios, mission, values, and statistics 3. **articles.html** - Articles listing page with category filtering and multiple articles 4. **subscribe.html** - Subscription page with detailed form, benefits, pricing tiers, and FAQ 5. **styles.css** - Shared stylesheet for all pages ### Navigation All pages include: - Fixed header with navigation menu - Links between all four main pages - Footer with additional navigation and resources - Responsive design that works on mobile, tablet, and desktop ## Key Features ### Home Page (index.html) - Hero section with call-to-action - 6 feature cards explaining newsletter content - Featured articles preview - Testimonials section - Subscribe CTA ### About Page (about.html) - Company story and history - Mission and values - Team member profiles (6 team members) - Coverage areas - Statistics dashboard - Subscribe CTA ### Articles Page (articles.html) - Category filter buttons (7 categories) - Featured article spotlight - 12 article cards with categories - Working JavaScript filter functionality - Load more button - Newsletter signup CTA ### Subscribe Page (subscribe.html) - Detailed benefits section - Comprehensive signup form with: - Name and email fields - Boat owner status selector - Interest checkboxes (8 options) - Location field - Terms and conditions - Subscription tier comparison (Free, Premium, Lifetime) - FAQ section - Form validation and success message ## Interactive Features ### JavaScript Functionality - Category filtering on articles page - Form validation on subscribe page - Success message display after form submission - Smooth scrolling navigation ### Responsive Design - Mobile-first approach - Breakpoints at 768px and 1024px - Grid layouts that adapt to screen size - Touch-friendly navigation ## Styling Highlights - Professional blue color scheme (#0066cc primary) - Modern gradient backgrounds - Smooth hover transitions - Card-based layouts with shadows - Accessible typography and contrast - Consistent spacing and alignment ## How to Deploy ### Quick Start 1. Upload all files to your web hosting service 2. Ensure index.html is in the root directory 3. Keep styles.css in the same directory as HTML files 4. No server-side code required - pure HTML/CSS/JavaScript ### Hosting Options - **GitHub Pages**: Free hosting, perfect for static sites - **Netlify**: Drag and drop deployment, free tier available - **Vercel**: Simple deployment with custom domains - **Traditional Web Hosting**: Any cPanel or FTP hosting works ### File Structure for Deployment ``` your-domain.com/ ├── index.html ├── about.html ├── articles.html ├── subscribe.html └── styles.css ``` ## Customization Guide ### Colors Main color variables are defined in styles.css: - Primary blue: #0066cc - Dark blue: #004c99 - Light blue: #00aaff - Success green: #28a745 - Background: #f8f9fa ### Content - Edit HTML files directly to change text, images, or structure - Article cards can be duplicated and modified - Team member sections can be expanded or reduced - Testimonials can be replaced with real ones ### Images Currently using emoji placeholders. To add real images: 1. Replace emoji in `.article-image` divs with `` tags 2. Add images for team members in `.team-photo` divs 3. Update featured article images ### Form Integration To connect the subscription form to your email service: 1. Add action attribute to form in subscribe.html 2. Integrate with services like Mailchimp, ConvertKit, or SendGrid 3. Update form field names to match your service requirements ## Browser Support - Chrome (latest) - Firefox (latest) - Safari (latest) - Edge (latest) - Mobile browsers (iOS Safari, Chrome Mobile) ## Performance - Minimal external dependencies - Optimized CSS with no unused styles - Fast loading times - No image optimization needed yet (using emoji placeholders) ## Next Steps 1.
